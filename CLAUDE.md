# CLAUDE.md — Team Balancer

Гайд для будущих сессий по проекту **Team Balancer** — инструменту визуализации и
балансировки команд кластера продуктовой разработки.

## Что это

Одностраничное клиентское приложение (один HTML-файл) для ИТ-лидера кластера: показывает
состав ~10 команд, позволяет перетаскивать участников между командами и подсвечивает
команды-выбросы по среднему грейду и доле подрядчиков. Без бэкенда, без сборки.

## ⛔️ Жёсткие ограничения (не нарушать)

1. **Закрытый финтех-контур** → один самодостаточный `index.html`, **без** CDN, npm,
   шага сборки и любых сетевых запросов. Новый код — только vanilla JS/CSS внутри файла.
2. **Реальные ФИО** в данных → ничего не уходит наружу; хранение только `localStorage` +
   ручной экспорт/импорт JSON.
3. **Без бэкенда.**

Подробное обоснование — в [docs/SPEC.md §2](docs/SPEC.md).

## Как запустить

Открыть `index.html` двойным кликом (`file://`) или раздать статик-сервером:
```
python3 -m http.server 8000   # затем http://localhost:8000/index.html
```
Сборки/установки нет.

## Архитектура (кратко)

- Состояние — модульные переменные `state / snapshots / baselineId / ui / history`.
- Рендер — ручной через `innerHTML` + делегирование событий на `#board`.
- **Конвенция:** любая мутация `state` завершается `commit()` = `pushHistory()` +
  `persist()` (localStorage) + `render()`. Чисто-UI изменения — `render()` без `commit()`.
- Карта блоков кода и DOM — в [SPEC §3.1 и §8](docs/SPEC.md).

## Проверки перед завершением правок

Синтаксис JS (нет тестового раннера, файл — HTML):
```bash
node -e "const fs=require('fs');const h=fs.readFileSync('index.html','utf8');fs.writeFileSync('/tmp/app.js',h.match(/<script>([\s\S]*?)<\/script>/)[1])" \
  && node --check /tmp/app.js && echo OK
```
Затем — прогон `docs/TEST_PLAN.md` через Playwright-агента.
