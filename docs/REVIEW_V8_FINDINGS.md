## Findings

### V8-F1 (P2): автотесты не полностью защищают `expiry` и strict-project policy для занятых позиций

**Где:**

- `docs/run-tests.js:1408` - `TC-17.13`.
- `docs/run-tests.js:1360` - `TC-17.8`.
- `docs/run-tests.js:1367` - `TC-17.9`.
- `docs/run-tests.js:1391` - `TC-17.12`.
- `docs/TEST_PLAN.md:350` - описание `TC-17.13`.
- `docs/SPEC.md:157-159`, `docs/SPEC.md:379-383` - `project/expiry` описаны как атрибуты ставки для всех позиций, не только вакансий.

**Что происходит:**

`TC-17.13` заявлен как проверка "Проект/срок задаются и занятой позиции", но фактически проверяет только:

- поля `#m_project/#m_expiry` видны у обычного сотрудника;
- `project` сохраняется;
- `.vac-chip.proj` появляется на карточке.

Сценарий не проверяет:

- сохранение `expiry` у занятой позиции;
- появление `.vac-chip.temp` у занятой позиции;
- нормализацию невалидного `expiry` для занятой позиции;
- сброс неизвестного `project` для занятой позиции;
- очистку `project` у занятой позиции при delete проекта из справочника.

Старые проверки `TC-17.8`, `TC-17.9`, `TC-17.12` по-прежнему сфокусированы на `find(p => p.isVacancy)` и `.chip.vacancy`, хотя модель уже расширена на любые позиции.

**Почему это риск:**

Реализация сейчас проходит ручную targeted-проверку, но регрессия в занятой позиции может пройти runner зеленым. Особенно рискованны обходные входы через import/localStorage: именно они раньше уже находили нарушения строгого инварианта `project === "" || projects.includes(project)`.

**Рекомендация:**

Расширить TS-17:

- В `TC-17.13` дополнительно установить `#m_expiry`, проверить `state.people.find(p => p.name === "Real").expiry` и `.vac-chip.temp`.
- Добавить или расширить кейс для import/localStorage: обычный сотрудник с `project: "Ghost"` получает `project === ""`.
- Добавить или расширить кейс для import/localStorage: обычный сотрудник с `expiry: "2026-99-99"` получает `expiry === ""`, `.vac-chip.temp` не показан.
- Расширить `TC-17.12`: delete проекта из справочника очищает `project` и у вакансии, и у занятой позиции; undo возвращает оба значения.
