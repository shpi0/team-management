## Открытые баги

### BUG-1 (P1) — фича «скрывать несовпадающих» обрушивает рендер доски
- **Кейс:** TC-10.4.
- **Симптом:** при активном фильтре/поиске включение `#filterMode` («скрывать несовпадающих»)
  убирает с доски **все** карточки, а не только несовпавшие. Доска остаётся пустой/полупустой.
- **Воспроизведение:**
  1. Любая фикстура с ≥1 совпадающим сотрудником (напр. поиск «S10», где есть человек «S10»).
  2. Ввести запрос в `#searchInput`.
  3. Поставить галочку «скрывать несовпадающих» (`#filterMode`).
  4. → На доске исчезают все `.chip`; в консоли `DOMException`.
- **Причина** — `index.html:542`:
  ```js
  if(matchFilter(p)) c.classList.add(ui.fHide?"":"hit");
  ```
  При `fHide=true` и совпавшем сотруднике вызывается `classList.add("")` →
  `DOMException: "The token provided must not be empty."`. Исключение прерывает цикл `render()`.
  Подтверждено через `page.on('pageerror')`.
- **Проверенный фикс** (прогнан на пропатченной копии — ошибок нет, совпавшие видны,
  несовпавшие `display:none`, TC-10.4 проходит):
  ```js
  if(matchFilter(p)){ if(!ui.fHide) c.classList.add("hit"); }
  ```
  Опционально почистить соседнюю строку 543: `c.classList.add(ui.fHide?"dim":"dim")` → `c.classList.add("dim")`.
- **Статус:** ✅ **исправлено 2026-06-30.** В `index.html` применён проверенный фикс (блок
  `// filter dim/hit` в `renderChip`): совпавший при `fHide` не получает пустого класса и остаётся
  видимым без подсветки; несовпавший — `dim` + `display:none`. Избыточная тернарная строка убрана.
  `node --check` — OK. TC-10.4 проходит.

## Новые фичи (2026-06-30) + фикс бага локаций — ✅ подтверждены прогоном (88/88)

Реализованы пользовательские фичи; тесты добавлены и прогнаны (Playwright, все зелёные).

- **Скрытие грейдов («глаз», `#gradeEye`):** маскирует индивидуальные грейды (`•••`), цвет роли
  сохраняется, агрегаты не маскируются, состояние персистентно. Тест **TC-15.1**.
- **Теги сотрудников:** `person.tags`, поле `#m_tags`, пилюли `.tag-pill` на карточке; в аналитике
  не участвуют. Тест **TC-15.2**.
- **Перестановка команд drag-and-drop:** грип `.team-grip[data-teamdrag]`, `reorderTeam`, порядок
  в `state.teams`, undo-обратимо. Тест **TC-15.3**.
- **Цвет квадрата грейда по роли:** `roleColor()` (явный `state.roleColors` или палитра по индексу),
  настройка в справочнике (`[data-rolecolor]`). Тесты **TC-15.4**, **TC-15.5**.

- **BUG-2 (P1) — полосы «Распределение по локациям» не заливались цветом.** Причина: `.barrow .fill`
  — inline `<span>`, для которого `height/width` игнорируются → высота полосы 0, заливки не видно.
  Фикс: `.barrow .fill{display:block;…}`. **Почему не поймали тесты:** старый TC-8.3 проверял лишь
  число баров и строку `width` в инлайн-стиле, а не вычисленную геометрию/цвет. TC-8.3 усилён
  (computed `height>0`, `width>0`, `display:block`, непрозрачный `background`).

Проверки: `node --check` для `index.html` и `docs/run-tests.js` — **OK**. Раннер вырос до **88**
кейсов — полный прогон **88/88**, регрессий нет (TC-2.9, TC-12.8, усиленный TC-8.3, TC-15.1–15.5 ✅).

**Правка тестов под новую разметку ролей (не приложения):** после добавления пикера цвета роли
