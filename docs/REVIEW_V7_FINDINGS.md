## Findings

### V7-F1 (P2): удаление проекта из UI оставляет ставки с удаленным проектом

**Где:**

- `index.html:1064` - generic delete для `[data-rm]` просто удаляет элемент из `state[key]`.
- `index.html:1062` - rename проекта мигрирует `people[].project`.
- `index.html:412-413` - импортная нормализация строго оставляет `project` только если он есть в `projects`.
- `docs/SPEC.md:363-368` - проект ставки описан как значение из справочника проектов.

**Что происходит:**

Состояние до действия:

```js
projects: ["Known"]
people: [
  { isVacancy: true, project: "Known", ... }
]
```

Шаги:

1. Открыть settings.
2. Удалить `Known` из `#s_projects` кнопкой `[data-rm]`.
3. Закрыть settings.

Фактическое состояние:

```json
{
  "projects": [],
  "project": "Known"
}
```

На доске ставка продолжает показывать чип `▣ Known`, хотя проекта уже нет в справочнике. При открытии модалки `<select id="m_project">` содержит только `— без проекта —`, выбранное значение становится пустым.

**Почему это риск:**

После V6 была выбрана строгая политика: ставка может ссылаться только на проект из `state.projects`; неизвестный проект из импорта сбрасывается в `""`. UI-delete проекта снова создает состояние, которое импорт уже запрещает:

- доска и модалка расходятся;
- проект может silently потеряться при следующем сохранении ставки;
- аналитика считает ставку проектной, хотя проекта нет в справочнике;
- инвариант `person.project === "" || state.projects.includes(person.project)` нарушается.

**Рекомендация:**

Для `key === "projects"` при удалении проекта очищать ссылки у ставок:

```js
$$("[data-rm]").forEach(b => b.onclick = () => {
  const [key, i] = b.dataset.rm.split(":");
  const removed = state[key][+i];
  state[key].splice(+i, 1);
  if (key === "projects") {
    for (const p of state.people) {
      if (p.project === removed) p.project = "";
    }
  }
  commit();
  rebind();
});
```

Это согласуется с V6-F2 strict policy. Если продуктово нужно сохранять "несправочные" проекты, тогда нужно осознанно поменять V6-F2 и UI select, но сейчас код/спека уже выбрали strict.

**Тесты:**

Добавить кейс:

- fixture со ставкой `project:"Known"` и `projects:["Known"]`;
- удалить проект через settings;
- проверить:
  - `state.projects` пустой;
  - `state.people.find(p => p.isVacancy).project === ""`;
  - `.vac-chip.proj` на доске не отображается;
  - undo возвращает проект и чип, если undo по справочникам должен работать как сейчас.
